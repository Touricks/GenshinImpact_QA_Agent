# 消融实验报告：Vector-Only vs KG+Vector

**日期**: 2026-02-18
**测试集**: basic (15 题, complexity 1-5)
**实验设计**: `docs_internal/design/2026-02-18_ablation_experiment_vector_only.md`

---

## 1. 实验概述

### 目的

评估知识图谱（KG）工具对 Agent v4 答案质量的贡献。通过 `ABLATION_MODE=vector_only` 禁用所有 KG 工具，仅保留 `search_memory`（向量搜索），与全量 7 工具 baseline 对比。

### 配置差异

| 项目 | with_KG (baseline) | without_KG (ablation) |
|------|-------------------|----------------------|
| 可用工具 | 7 (lookup_knowledge, find_connection, get_timeline, trace_causality, compare_entities, explore_subgraph, search_memory) | 1 (search_memory) |
| System Prompt | 标准 KG-First 策略 | 追加消融提示，禁用 KG-First 规则 |
| LLM | gemini-2.5-flash | gemini-2.5-flash |
| Workers | 3 | 3 |

## 4. Faithfulness 下降分析

8/15 题在 vector_only 模式下 faithfulness 从 1.0 降至 0.8：

### 4.1 下降原因分类

| 类型 | 描述 | 受影响题目 |
|------|------|-----------|
| **推理型幻觉** | 检索文本未直接包含答案，LLM 基于间接线索推理补全，导致事实偏差 | NK-01, NK-06 |
| **精度不足** | 缺少 KG 结构化属性字段（role / political_status / relationship），LLM 只能从叙事段落推断 | NK-02, NK-07, NK-22 |
| **信息碎片化** | 复杂叙事需跨多段落综合，vector 检索只返回片段，LLM 补全引入不确定声明 | NK-15, NK-36, NK-41 |

### 4.2 未下降但答案实质偏差的题目

| ID | 自评分数 | 问题 |
|----|---------|------|
| **NK-09** | faith=1.0 (两组均为 1.0) | with_KG→"魇夜之莺队长"(正确); without_KG→"灯塔守护者"(与 golden answer 不符) |

NK-09 暴露了 **自评 grader 的盲点**：grader 评估"答案是否忠实于检索结果"，而非"检索结果是否为正确信源"。vector 检索到角色侧写文件（task:10000127）中的次要身份描述，LLM 忠实引用了该描述，因此 faithfulness=1.0——但答案实际偏离了 golden answer。这类错误只有通过 LLM-as-Judge 准确度评估（eval_accuracy.py）才能检测。

## 5. 典型案例分析

### 5.1 NK-01: 实体属性精确查询

**问题**: Nod-Krai 区域目前名义上隶属于哪个国家？
**Golden Answer**: 无隶属状态，处于高度自治

| | with_KG | without_KG |
|---|---------|-----------|
| 工具 | `lookup_knowledge(entity="Nod-Krai")` | `search_memory` ×2 |
| 关键信息源 | KG 实体属性 `political_status: "成立联合会进行自治"` | 叙事段落中提及愚人众驻军、"Krai"词源 |
| 答案 | "不隶属于提瓦特七国中的任何一国，实行**自治**" | "名义上隶属于**至冬国**" |
| 与 Golden Answer | **一致** | **矛盾** |
| Faithfulness | 1.0 | 0.8 |
| Duration | 24s | 48s |

**分析**: KG 的结构化属性字段直接给出"自治"这一精确答案。vector 搜索只返回叙事段落，其中愚人众活动频繁 + 俄语词源"Krai"（边疆区），LLM 据此推理出"隶属至冬国"——这是一个合理但错误的推断。

### 5.2 NK-09: 检索源错误（Grader 盲点）

**问题**: 叶洛亚在执灯人军团中担任什么职位？
**Golden Answer**: 调查分队「魇夜之莺」的队长

| | with_KG | without_KG |
|---|---------|-----------|
| 工具 | `lookup_knowledge(entity="叶洛亚")` ×2 | `search_memory` ×1 |
| 关键信息源 | KG 实体 `role: "调查分队队长"`, `affiliation: "魇夜之莺"` | 角色侧写文件 (task:10000127) "纳德里赫特的灯塔守护者" |
| 答案 | "「魇夜之莺」调查分队的**队长**" | "主要担任**灯塔守护者**" |
| 与 Golden Answer | **一致** | **不符** |
| Faithfulness | 1.0 | 1.0 (!) |
| Duration | 15s | 25s |

**分析**: vector 检索按 embedding 相似度排序，角色侧写文件（描述叶洛亚日常生活）的 dense score 高于主线剧情对话。without_KG 的 LLM 忠实引用了这段侧写文本，因此 faithfulness grader 给出 1.0。但实际答案偏离了叶洛亚在主线中的核心身份。KG 的 `role` 结构化字段完全消除了这种歧义。

**启示**: 自评体系的 faithfulness 指标只能检测"是否忠实于检索结果"，无法检测"检索结果本身是否为正确信源"。需要 golden answer 对比评估来补充。

### 5.3 NK-36: 复杂叙事的工具效率

**问题**: 详细梳理从旅行者抵达挪德卡莱到彻底击败"教授"的执灯人主线完整时间线
**Complexity**: 5

| | with_KG | without_KG |
|---|---------|-----------|
| Tool Calls | 10 | 12 |
| Duration | 85s | 97s |
| Faithfulness | 1.0 | 0.8 |
| Completeness | 0.8 | 1.0 |

**分析**: 此题需要跨越多章节的时间线综合。with_KG 使用 `get_timeline` 和 `explore_subgraph` 等结构化工具高效获取事件链；without_KG 则需要多次 `search_memory` 用不同关键词搜索，但仍有信息遗漏导致 LLM 补全引入不确定声明，faithfulness 降至 0.8。有趣的是 without_KG 的 completeness 反而更高（1.0 vs 0.8），可能因为多次搜索覆盖了更多细节片段。

## 6. 运行时长对比

### 按复杂度分组

| Complexity | with_KG Avg | without_KG Avg | Δ |
|-----------|------------|---------------|---|
| 1 (5题) | 28.0s | 36.8s | +8.8s |
| 2 (2题) | 43.5s | 47.0s | +3.5s |
| 4 (3题) | 52.4s | 51.1s | -1.3s |
| 5 (5题) | 61.8s | 73.3s | +11.5s |

简单题（complexity 1）和复杂题（complexity 5）vector_only 明显更慢。中等题（complexity 2-4）差异不大。

### 原因
- 简单题：KG 工具一次 `lookup_knowledge` 即可获得精确答案（~15-25s），vector 需要多轮搜索确认（~25-48s）
- 复杂题：KG 工具（timeline/subgraph）直接返回结构化事件链，vector 需要多轮关键词搜索拼凑

---

## 7. 结论

### 7.1 KG 工具的核心价值

1. **精确属性查询**: `lookup_knowledge` 直接返回实体的结构化属性（political_status, role, affiliation），消除 LLM 推理偏差（NK-01, NK-09）
2. **关系链追溯**: `find_connection`, `explore_subgraph` 高效获取实体间关系网络，避免多轮搜索
3. **时间线构建**: `get_timeline` 直接返回事件序列，复杂叙事题效率更高（NK-36）
4. **检索源可靠性**: KG 数据来自策划审核后的结构化知识库，避免角色侧写/非主线文本的检索污染

### 7.2 Vector-Only 的表现

- **Pass Rate 不变**: 15/15 全部通过，说明 vector 搜索在"能否回答"维度上足够
- **叙事综合类表现接近**: NK-12, NK-13, NK-19, NK-32, NK-39 等叙事分析题 faithfulness 未下降
- **更长的响应时间**: 平均多 ~7s，主要因为需要更多搜索轮次

### 7.3 自评体系局限性

NK-09 案例揭示了自评 faithfulness grader 的盲点：它只评估"答案是否忠实于检索结果"，但当检索结果本身来源错误时（角色侧写 vs 主线剧情），grader 无法检测。**建议**: 在关键评估中配合 `eval_accuracy.py` (LLM-as-Judge + golden answer) 进行交叉验证。

### 7.4 总结

KG+Vector 全量模式在 faithfulness (+10.7%) 和响应速度 (-15%) 上明显优于 vector_only。**KG 工具在精确事实查询场景不可替代**，尤其是实体属性和关系查询。但对于开放式叙事综合题，vector 搜索已能提供足够的上下文信息。
